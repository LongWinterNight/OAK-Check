# Git workflow

## Главное правило

**После каждого изменения — коммит + пуш в `origin/main`.** Сообщения коммитов — **на русском**.

Источник: feedback от пользователя. Причина: пользователь явно попросил всё заносить в GitHub с коммитами, чтобы видеть историю.

## Репозиторий

- URL: https://github.com/LongWinterNight/OAK-Check.git
- Ветка: `main`
- Рабочая директория: `d:\AI\Oak3CRM\oak-check`

## Процесс

```bash
# 1. проверить что меняется
git status

# 2. добавить конкретные файлы (не `git add .`, чтобы не зацепить .env и т.п.)
git add <file1> <file2>

# 3. коммит с русским сообщением (HEREDOC для многострочных)
git commit -m "$(cat <<'EOF'
feat(plan-12): адаптация ChaptersPanel под мобилу

Добавлены media queries, drill-down состояние через URL state.
EOF
)"

# 4. пуш
git push origin main
```

## Формат сообщений

```
<type>(<scope>): <короткое описание на русском>

<опциональное подробное описание на русском>
```

**Типы:** `feat`, `fix`, `docs`, `refactor`, `perf`, `style`, `test`, `chore`.

**Scope (типичные):**
- `plan-12` / `plan-XX` — относится к плану
- `mobile` / `tablet` — адаптив
- `auth`, `api`, `db`, `ui` — область
- `theme` — тема/акценты

**Примеры из истории проекта:**
- `fix(theme): oak теперь дефолтный акцент, убран баг с фиолетовой палитрой`
- `fix(tablet): переполнение таблицы в настройках + фикс отступов`
- `fix(mobile): переполнение тулбара, горизонтальный скролл таблиц`
- `feat(plan-12): мобильный и планшетный адаптив UI — Plan 12.2 завершён`

## Что НЕ делать

- **Не амендить** (`git commit --amend`) уже запушенный коммит — создаём новый коммит.
- **Не использовать `--no-verify`** — хуки (если есть) не пропускать.
- **Не использовать `git add .` / `git add -A`** — можно случайно зацепить `.env.local`, большие бинари.
- **Не пушить с `--force` в `main`** — защищённая ветка.
- **Не игнорировать CRLF-предупреждения** — это ожидаемо на Windows, git нормализует.

## Если пре-коммит хук падает

1. Смотрим ошибку, чиним
2. `git add` поправленные файлы
3. Новый `git commit` (НЕ `--amend`)

## Проверка перед пушем

Если меняли TS/TSX:
- [ ] `npm run lint` (опционально, если CI всё равно прогонит)
- [ ] `npm run build` локально при крупных рефакторингах

Адаптив (чистый CSS):
- [ ] Проверено в DevTools на 360/768/1024 — см. [coding-standards.md](./coding-standards.md)

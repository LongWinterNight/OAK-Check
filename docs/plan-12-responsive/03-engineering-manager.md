# 03 — Engineering Manager

## Стандарты кода для Plan 12

### CSS

1. **Не вводить новых брейкпоинтов.** Только:
   - `(max-width: 767px)` — mobile
   - `(min-width: 768px) and (max-width: 1023px)` — tablet
   - desktop — default
2. **Не хардкодить пиксели отступов.** Использовать `var(--spacing-*)` из [globals.css](../../app/globals.css).
3. **Не дублировать селекторы между модулями.** Если одно правило повторяется в 3+ местах — выносим утилиту в `globals.css`.
4. **Не использовать `!important`.** Исключение — override сторонних библиотек, с комментарием-причиной.
5. **Сначала desktop, потом overrides.** Media queries — в конце файла модуля, после базовых правил.

### React

1. **Никаких `useWindowSize` хуков для рендер-ветвления.** Адаптив — через CSS. Исключение: заведомо разные компоненты (BottomNav vs Sidebar) — тогда рендерим оба, прячем через CSS `display: none`.
2. **`touch-action: manipulation`** на кликабельных элементах, чтобы убрать 300ms delay на тапе.
3. **Модалки на мобиле — full-screen или bottom-sheet.** Центрированное окно с бэкграундом ≤ 400px — только на десктопе.

### Git / PR

- Коммит после каждого изменения + push в `main` (см. [feedback_git_commits](../../../memory/feedback_git_commits.md))
- Сообщение коммита формата: `feat(plan-12): <scope> — <short description>` или `fix(mobile|tablet): ...`
- В сообщении упоминать затронутый экран/компонент

## Ревью-чеклист

Перед merge проверить:

- [ ] Изменения протестированы на 360 / 768 / 1024 / 1440 в DevTools
- [ ] Нет horizontal scroll
- [ ] Touch-targets ≥ 44×44 на мобиле
- [ ] Focus-ring виден и на мобиле (не перекрывается клавиатурой)
- [ ] Скролл-контейнеры имеют `overflow-y: auto` + `overscroll-behavior: contain` где нужно
- [ ] Нет новых `!important` без комментария
- [ ] Нет `useWindowSize`/аналогов JS-рендеринга по ширине

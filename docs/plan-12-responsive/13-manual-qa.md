# 13 — Manual QA Engineer

## Цель

Пройти реальными пальцами по реальным устройствам. Автоматика не ловит тактильное «ой, кнопка маленькая» и iOS-specific баги.

## Device matrix

Минимальный набор (покрывает ~95% мобильной аудитории на 2026):

| Устройство | OS/Browser | Приоритет | Комментарий |
|---|---|---|---|
| iPhone 13/14/15 | iOS 17, Safari | **P0** | Main mobile target |
| iPhone SE (2nd gen) | iOS 16, Safari | P1 | 375×667 — самый узкий актуальный |
| Android mid-range (Samsung A5x) | Chrome | **P0** | Main Android target |
| iPad (10-го gen или Air) | iOS 17, Safari | **P0** | Main tablet target |
| Android tablet (Lenovo P11 class) | Chrome | P2 | Меньше юзеров, но бывают |

Плюс DevTools эмуляция как разминка — но **финальная приёмка на реальных устройствах**.

## Чеклист по ролям

Для каждой роли — залогиниться с соответствующим аккаунтом и пройти.

### ARTIST на iPhone 13

- [ ] Логин с email/password через мобильную клавиатуру
- [ ] Open dashboard — MyDay, MyShots читабельны без зума
- [ ] Tap на шот → checklist открывается
- [ ] На мобильном drill-down: глава → пункты → панель работают, back-кнопка возвращает корректно
- [ ] Отметить пункт чеклиста — чекбокс реально попадает пальцем
- [ ] Inline-редактирование пункта — клавиатура не перекрывает поле
- [ ] Оставить комментарий — keyboard не ломает layout, scroll к последнему сообщению
- [ ] Сделать фото камерой → загрузка в шот
- [ ] Выход

### ARTIST на Android (Samsung A5x)

- [ ] То же, что на iPhone + проверить back-button системный (жест/кнопка) — уходит на уровень вверх в drill-down

### LEAD на iPad

- [ ] Dashboard — виджеты в tablet-layout
- [ ] Kanban — tap удерживать шот → перетащить в соседнюю колонку (touch DnD)
- [ ] Kanban — свайп между колонками
- [ ] Projects → проект → шот → назначить исполнителя из списка
- [ ] Создать новый проект через `NewProjectModal` (full-screen на мобиле, модалка на планшете)

### QA на iPad

- [ ] `/review` — список в удобном виде
- [ ] Открыть шот, переключить статус
- [ ] Открыть рендер, добавить пин-комментарий (касанием на нужную точку)
- [ ] Закрыть рендер, вернуться

### PM на iPad

- [ ] Dashboard — DeadlineAlert виден и кликабелен
- [ ] Открыть `/projects`, фильтр, создать проект
- [ ] Редактировать существующий проект

### ADMIN на iPad

- [ ] `/admin` — статусы и компактная аналитика читабельна
- [ ] `settings/tabs/TeamTab` — инвайт нового юзера (input email ок, picker ролей ок)
- [ ] CSV-export — кнопка есть, но можно пометить «открыть с десктопа» — поведение допустимое

## Cross-cutting проверки (на каждом устройстве)

- [ ] Нет горизонтального скролла
- [ ] Нет «обрезанных» кнопок/текста
- [ ] Focus-ring виден при tab-навигации (актуально для iPad с bluetooth клавиатурой)
- [ ] Темная тема переключается, все экраны читабельны
- [ ] Смена акцентного цвета применяется без перезагрузки (см. [AppearanceTab](../../app/(app)/settings/tabs/AppearanceTab.tsx))
- [ ] Клавиатура не перекрывает активное поле ввода
- [ ] В standalone (добавлено на домашний экран) — bottom nav не залезает под home indicator
- [ ] Landscape на iPhone — layout не ломается

## Отчёт

Завершение device matrix — скриншоты проблемных мест + список дефектов в формате:

```
[device] [role] [screen] [severity P0/P1/P2] — что не так
```

Баги P0 возвращаем в Frontend (06) как срочные, P1 — следующим спринтом, P2 — в отдельный Plan.

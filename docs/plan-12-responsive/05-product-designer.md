# 05 — Product Designer (UI/UX)

## Ревизия системы под мобилу

### Touch targets

Минимум **44×44 px** (Apple HIG) / **48×48 px** (Material) для всего интерактивного на мобиле.

Проверить и при необходимости поправить:

- [ ] `Button` размеры — не допускаем sm-кнопок < 36px на мобиле
- [ ] `Check3` чекбокс — на мобиле увеличить область клика (можно через `::before` с отрицательными отступами)
- [ ] `Segmented` — таб ≥ 44px высоты
- [ ] Иконки-кнопки в тулбарах — padding минимум обеспечивающий 44×44 area
- [ ] Closeable ×-кнопки в модалках/тегах — 32px мало, расширить на мобиле

### Spacing на мобиле

Глобальные `--spacing-*` уже есть. Правило для мобилы:

- Отступы секций/страниц: **12px** (не 24px как на десктопе) — уже применено в library/activity
- Промежутки между карточками: `--spacing-3` (8px), не `--spacing-4`
- Padding модалок: `--spacing-4` (14px) вместо `--spacing-5/6` на мобиле

### Типографика

На мобиле **не уменьшать** размеры ниже десктопа. Проверить, что базовый текст ≥ 14px (лучше 15). Заголовки — можно уменьшать (h1 с 24 до 20).

- [ ] Проверить `ShotHeader` на мобиле — не «схлопнутая ли» типографика
- [ ] Проверить `MyDay`/`MyShots` виджеты — не мелкий ли подзаголовок

### Модалки на мобиле

**Правило:** модалка на мобиле ≠ центрированное окно. Паттерны:

1. **Full-screen modal** — для сложных форм (`NewProjectModal`, `UploadRenderModal`, `ApplyTemplateModal`)
2. **Bottom sheet** — для коротких действий (`ConfirmDialog`, выбор опции)
3. **Stay centered** — только для `Toast`, мелких подтверждений на планшете

На десктопе/планшете паттерны сохраняются как есть.

### Компоненты-специфика

#### `DatePicker`
- Mobile: full-width popover, якорится снизу к инпуту, не справа
- Trigger-кнопка ≥ 44px высоты

#### `ChaptersPanel` / `ItemsList` / `RightPanel`
- На мобиле три панели в ряд **не работают** — нужен таб-свитч (Главы / Пункты / Панель) или последовательный drill-down (глава → пункты → панель через хлебные крошки)
- Рекомендация: drill-down с хлебными крошками, т.к. проще для миграции

#### `KanbanBoard`
- На мобиле — horizontal swipe между колонками, одна колонка на экране
- Drag-and-drop touch через `pointer-events` — проверить библиотеку

#### `CommentsPanel`
- Input снизу, липкий к safe-area
- При фокусе на мобиле — ensure scroll to bottom, чтобы клавиатура не перекрывала

### Dark mode на мобиле

Проверить, что при OLED-чёрном фоне на мобиле нет «серых пятен» (неверный `--surface-*`). Уже существующие переменные темы (`[data-theme="dark"]`) должны быть достаточны.

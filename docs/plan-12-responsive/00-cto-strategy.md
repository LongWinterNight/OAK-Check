# 00 — CTO

## Цель Plan 12

Довести OAK·Check до состояния, когда **любая CRM-роль может закрыть свой дневной сценарий с мобильного устройства** без переключения на десктоп.

## KPI завершения

- **Lighthouse Mobile ≥ 85** на главных страницах (dashboard, projects, checklist, kanban)
- **LCP < 2.5s** на 4G-эмуляции (Moto G Power класс)
- **0 горизонтального скролла** на 360px ширине на всех экранах кроме явно-desktop-only (dashboard ADMIN-аналитика)
- **100% touch-targets ≥ 44×44 px** на мобиле (WCAG 2.5.5)

## Стратегические решения

1. **PWA-first, не нативка.** Устанавливаемый web app + офлайн базовый — да. React Native — нет (вне ресурсов).
2. **Desktop-first CSS, overrides через media queries.** Не переписываем в mobile-first, это удвоит работу.
3. **Один дизайн-язык на всех ширинах.** Никаких «мобильных версий» с отдельными компонентами — только адаптированные состояния существующих.
4. **Роль ADMIN — исключение.** Админские дашборды и CSV — разрешено требовать планшет 768+. Не тратим ресурсы на сжатие аналитических таблиц под 360px.

## Риски

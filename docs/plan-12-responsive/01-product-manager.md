# 01 — Product Manager

## Продуктовый контекст

Мобильный use-case в CRM различается **радикально по роли**. Адаптив не равен «всё сжать» — для каждой роли свой набор priority-экранов.

## User stories по CRM-ролям

### ARTIST (высший приоритет)
> Я художник. Я хочу со смартфона быстро снять рендер, загрузить его в шот, оставить комментарий и отметить пункт чеклиста выполненным, не доставая ноутбук.

**Priority screens (mobile):**
1. `UploadSheet` (уже есть) — проверить полноту
2. `checklist/[shotId]` — чеклист + комментарии + рендер в одной высоте
3. `ItemsList` — inline-редактирование, чекбоксы
4. `CommentsPanel` — комфортный ввод с клавиатуры
5. Активность — feed последних действий

### LEAD (высокий)
> Я тимлид. Мне нужно с планшета оценить прогресс команды, перекинуть шот между статусами на канбане, и назначить исполнителя.

**Priority screens (tablet):**
1. `kanban` — drag-and-drop работает на touch
2. `dashboard` — MyShots, MyDay виджеты
3. `projects/[projectId]` — список шотов + массовые действия
4. Назначение исполнителя (assign) — доступно с мобилы

### QA (средний)
> Я QA. Я хочу с планшета пройтись по ревью-очереди, поменять статус шота, оставить комментарий с пином на рендере.

**Priority screens (tablet):**
1. `review` — список на ревью
2. `checklist` с открытым `RenderPreview` + `CommentsPanel`
3. Пины на рендере (из Plan 09) — касание пальцем работает

### POST (средний)
> Я пост-продакшен. Загрузить финальный рендер, отметить этап.

**Priority screens (mobile):**
- Те же, что у ARTIST, с фокусом на загрузку

### PM (средний)
> Я PM. Мне нужно с планшета создать проект, посмотреть дедлайны и увидеть риск-алерт.

**Priority screens (tablet):**
1. `dashboard` с `DeadlineAlert`
2. `NewProjectModal`
3. `projects` — фильтры
4. Редактирование проекта

### ADMIN (низкий для mobile, высокий для tablet)
> Я админ. Аналитику смотрю с десктопа, но раз в день с планшета хочу проверить статус хранилища и добавить инвайт.

**Priority screens (tablet):**
1. `admin` — статусы, аналитика компактно
2. `settings/tabs/TeamTab` — инвайт с планшета
3. ❌ CSV-экспорт и тяжёлая аналитика — остаются desktop-only

## Приоритизация задач

**P0 (блокирует mobile-ready):** ARTIST flow — чеклист + комментарии + upload
**P1 (блокирует tablet-ready):** LEAD/QA flow — канбан-touch + review + assign
**P2 (nice-to-have):** PM-модалки, ADMIN планшетный режим
**P3 (после Plan 12):** Offline, push-уведомления, нативные жесты
